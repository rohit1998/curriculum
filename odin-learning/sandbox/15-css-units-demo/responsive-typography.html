Different approaches to responsive typography using viewport units, showing the problems and solutions from the CSS-Tricks article.

‚ùå BAD: Direct Viewport Units (font-size: 5vw)

This text scales too dramatically! It goes from unreadably small to huge very quickly. This is why the article said direct viewport units are "dangerous" for typography.

‚úÖ GOOD: Base Size + Viewport (font-size: calc(16px + 1vw))

This text has a minimum base size of 16px and grows 1px for every 100px of viewport width. Much more controlled and readable at all sizes!

‚ú® BETTER: Different Growth Rates

This heading grows faster (calc(1.2em + 3vw))

This body text grows slower (calc(16px + 0.5vw)) and even has responsive line-height (calc(1.1em + 0.5vw)). This creates a more dynamic typographic scale on larger screens while keeping everything readable on mobile. Notice how the heading grows more dramatically than this paragraph, creating better hierarchy.

The heading is also constrained with a media query at 50em width to prevent it from becoming too large. This technique gives you "no media-queries required" responsive typography with just a single breakpoint for constraints.

üîç Key Insights:

  • Different elements can have different growth rates - headings vs body text
  • One media query can constrain maximum sizes when needed
  • Line-height can also be responsive for better readability
