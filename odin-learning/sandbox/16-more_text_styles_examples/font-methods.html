Font Implementation Methods

1. System Font Stack

This text uses the system font stack. It will look native on each operating system - San Francisco on macOS, Segoe UI on Windows, and Ubuntu on Linux.

/* CSS */ .system-font { font-family: system-ui, "Segoe UI", Roboto, Helvetica, Arial, sans-serif; }

Why use system fonts?

  • ✅ Performance: No download time - fonts are already installed
  • ✅ Consistency: Matches the user's OS interface
  • ✅ Reliability: Always available
  • ✅ Privacy: No external requests

2. Google Fonts (Online Library)

This text uses Open Sans from Google Fonts. It's clean, readable, and widely used.

Method 1: Link in HTML head

<link rel="preconnect" href="https://fonts.googleapis.com"> <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin> <link href="https://fonts.googleapis.com/css2?family=Open+Sans:wght@300;400;700&display=swap" rel="stylesheet">

Method 2: @import in CSS

/* CSS */ @import url('https://fonts.googleapis.com/css2?family=Open+Sans:wght@300;400;700&display=swap');

Then use in CSS:

.google-font { font-family: 'Open Sans', sans-serif; /* Always include fallback! */ }

Considerations:

  • ⚠️ Performance: Requires download time
  • ⚠️ Privacy: GDPR concerns with Google Fonts
  • ✅ Design: Wide variety of professional fonts
  • ✅ Consistency: Same font across all devices

3. Self-Hosted Fonts

This text would use a self-hosted font (falls back to system font since we don't have the actual font file).

/* CSS */ @font-face { font-family: 'LocalFont'; src: url('./fonts/local-font.woff2') format('woff2'), url('./fonts/local-font.woff') format('woff'); font-weight: normal; font-style: normal; font-display: swap; /* Shows fallback while loading */ } .local-font { font-family: 'LocalFont', system-ui, sans-serif; }

Font formats (in order of preference):

  • WOFF2: Best compression, modern browsers
  • WOFF: Good compression, wider support
  • TTF/OTF: Fallback for older browsers

Benefits:

  • ✅ Control: You own the font files
  • ✅ Privacy: No external requests
  • ✅ Performance: Can optimize loading
  • ⚠️ Setup: More complex implementation

Best Practices

  • Always include fallbacks in your font-family stack
  • Use preconnect for external font libraries
  • Consider font-display: swap for better loading experience
  • Limit font weights and styles to reduce load time
  • Test on different devices to ensure fallbacks work well
